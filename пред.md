# Преддипломная работа

## Название темы

**"Разработка веб-приложения для каталогизации и управления экспонатами художественного факультета"**

## Обоснование выбора темы

В современном образовательном процессе возникает необходимость в цифровизации и систематизации студенческих работ. Создание веб-приложения для каталогизации экспонатов позволит упростить процесс учета, хранения и представления художественных работ студентов, обеспечив удобный доступ к информации как для преподавателей, так и для студентов.

## Цель работы

Разработать веб-приложение для каталогизации и управления экспонатами художественного факультета ВГУ имени П.М. Машерова с возможностью просмотра каталога, детальной информации об экспонатах и базовой административной панелью для управления контентом.

## Задачи преддипломной работы

1. Проанализировать требования к системе каталогизации экспонатов
2. Разработать архитектуру веб-приложения
3. Спроектировать структуру базы данных для хранения информации об экспонатах
4. Реализовать пользовательский интерфейс для просмотра каталога экспонатов
5. Реализовать страницы детального просмотра экспонатов
6. Разработать административную панель для управления экспонатами (CRUD операции)
7. Реализовать систему загрузки и хранения изображений экспонатов
8. Обеспечить базовую фильтрацию и поиск по каталогу

## Что выносится на преддипломную работу

### 1. Основная структура проекта
- Настройка Next.js 14 с TypeScript
- Конфигурация Tailwind CSS для стилизации
- Базовая структура папок и компонентов
- Настройка Prisma ORM с SQLite базой данных

### 2. База данных
- Модель `Exhibit` с основными полями:
  - Основная информация: id, title, description, fullDescription
  - Метаданные: category, year, creationDate
  - Информация об авторе: studentName, studentCourse, studentGroup
  - Информация о руководителе: supervisor, supervisorPosition, supervisorRank, supervisorDepartment
  - Дополнительные поля: dimensions, currentLocation, inventoryNumber
  - Медиа: previewImage, images (JSON массив)
  - Флаги: isPublic
- Миграции базы данных

### 3. Публичная часть (Frontend)

#### 3.1. Главная страница (`app/page.tsx`)
- Hero-секция с приветствием
- Блок с популярными работами (топ-6 экспонатов)
- Статистика (количество работ)
- Навигационные ссылки на каталог и о проекте

#### 3.2. Каталог экспонатов (`app/catalog/page.tsx`)
- Список всех публичных экспонатов в виде карточек
- Поиск по названию, описанию, категории, автору
- Фильтрация по:
  - Категории (живопись, скульптура, графика и т.д.)
  - Году создания
- Пагинация результатов (6 экспонатов на страницу)
- Сортировка экспонатов

#### 3.3. Страница экспоната (`app/exhibit/[id]/page.tsx`)
- Детальная информация об экспонате:
  - Название, описание, полное описание
  - Галерея изображений
  - Информация об авторе и руководителе
  - Дополнительные характеристики (размеры, местонахождение)
- Навигация между экспонатами (предыдущий/следующий)

#### 3.4. Страница "О проекте" (`app/about/page.tsx`)
- Информация о проекте
- Описание функциональности

### 4. Компоненты

#### 4.1. Общие компоненты
- `Navbar.tsx` - навигационное меню
- `Footer.tsx` - подвал сайта
- `ExhibitCard.tsx` - карточка экспоната для каталога
- `SafeImage.tsx` - безопасный компонент для отображения изображений

#### 4.2. Компоненты админ-панели
- `ExhibitForm.tsx` - форма создания/редактирования экспоната
- `ExhibitList.tsx` - список экспонатов в админ-панели
- `PageContentEditor.tsx` - редактор контента страниц (базовая версия)

### 5. Административная панель (`app/admin/page.tsx`)

#### 5.1. Аутентификация
- Простая система входа (логин/пароль)
- Два уровня доступа: admin и super-admin
- Хранение сессии в localStorage и cookies

#### 5.2. Управление экспонатами
- Просмотр списка всех экспонатов (включая приватные)
- Создание нового экспоната
- Редактирование существующего экспоната
- Удаление экспоната
- Загрузка изображений (превью и галерея)
- Управление видимостью (публичный/приватный)

#### 5.3. Редактор контента страниц (только для super-admin)
- Редактирование текстового контента главной страницы
- Сохранение изменений в базу данных

### 6. API маршруты (`app/api/`)

#### 6.1. Экспонаты
- `GET /api/exhibits` - получение списка экспонатов
  - Поддержка фильтрации по isPublic
  - Поддержка админ-режима (показ всех экспонатов)
- `GET /api/exhibits/[id]` - получение одного экспоната
- `POST /api/exhibits` - создание нового экспоната
- `PUT /api/exhibits/[id]` - обновление экспоната
- `DELETE /api/exhibits/[id]` - удаление экспоната

#### 6.2. Загрузка файлов
- `POST /api/upload` - загрузка изображений
  - Валидация типов файлов
  - Сохранение в `public/images/`
  - Возврат пути к загруженному файлу

#### 6.3. Контент страниц
- `GET /api/page-content` - получение контента страниц
- `PUT /api/page-content` - обновление контента (только для super-admin)

#### 6.4. Статические файлы
- `GET /api/storage/[...path]` - обслуживание загруженных файлов

### 7. Утилиты и библиотеки (`lib/`)

- `prisma.ts` - инициализация Prisma Client
- `exhibits.ts` - функции для работы с экспонатами:
  - `getExhibits(isAdmin: boolean)` - получение списка
  - `getExhibitById(id: string, isAdmin: boolean)` - получение по ID
- `pageContent.ts` - функции для работы с контентом страниц:
  - `getPageContent()` - получение контента
  - `savePageContent(content: PageContent)` - сохранение контента

### 8. Типы TypeScript (`types/`)

- `exhibit.ts` - интерфейс `Exhibit`
- `pageContent.ts` - интерфейс `PageContent`
- `global.d.ts` - глобальные типы

## Технологический стек

- **Frontend Framework**: Next.js 14 (App Router)
- **Язык программирования**: TypeScript
- **Стилизация**: Tailwind CSS
- **База данных**: SQLite
- **ORM**: Prisma
- **Деплой**: Docker (опционально)

   - Оптимизация производительности для мобильных устройств
