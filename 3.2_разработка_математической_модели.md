# 3.2 Разработка математической модели

Математическая модель системы описывает основные операции и алгоритмы работы с данными.

## 3.2.1 Модель поиска и фильтрации

Поиск экспонатов осуществляется по множеству критериев:

Пусть E — множество всех экспонатов, E = {e₁, e₂, ..., eₙ}

Для каждого экспоната eᵢ определены атрибуты:
- title(eᵢ) — название
- description(eᵢ) — описание
- category(eᵢ) — категория
- studentName(eᵢ) — имя автора
- supervisor(eᵢ) — научный руководитель
- year(eᵢ) — год создания

**Функция поиска:**

Для запроса q функция поиска возвращает множество экспонатов:

Search(q) = {eᵢ ∈ E | q ⊆ (title(eᵢ) ∪ description(eᵢ) ∪ category(eᵢ) ∪ studentName(eᵢ) ∪ supervisor(eᵢ))}

где q — множество ключевых слов запроса.

**Функция фильтрации:**

Filter(E, C, Y) = {eᵢ ∈ E | category(eᵢ) ∈ C ∧ year(eᵢ) ∈ Y}

где:
- C — множество выбранных категорий
- Y — множество выбранных годов

## 3.2.2 Модель позиционирования в 3D пространстве

Каждый экспонат в виртуальной галерее имеет позицию в трехмерном пространстве:

P(eᵢ) = (xᵢ, yᵢ, zᵢ)

где:
- xᵢ = galleryPositionX(eᵢ)
- yᵢ = galleryPositionY(eᵢ)
- zᵢ = galleryPositionZ(eᵢ)

**Трансформация экспоната:**

T(eᵢ) = (P(eᵢ), S(eᵢ), R(eᵢ))

где:
- S(eᵢ) = galleryScale(eᵢ) — масштаб
- R(eᵢ) = (0, galleryRotationY(eᵢ), 0) — поворот по оси Y

**Расстояние между экспонатами:**

Для предотвращения наложения экспонатов вычисляется расстояние:

d(eᵢ, eⱼ) = √((xᵢ - xⱼ)² + (yᵢ - yⱼ)² + (zᵢ - zⱼ)²)

## 3.2.3 Модель пагинации

Для отображения большого количества экспонатов используется пагинация:

Пусть pageSize = 6 — количество экспонатов на странице.

Для страницы p множество экспонатов:

Page(p, E) = {eᵢ ∈ E | (p - 1) × pageSize < i ≤ p × pageSize}

Общее количество страниц:

TotalPages(E) = ⌈|E| / pageSize⌉
